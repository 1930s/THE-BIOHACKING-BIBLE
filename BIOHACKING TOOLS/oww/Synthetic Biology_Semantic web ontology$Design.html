==Registry use cases== *Subpart Search: search for parts that match a portion of this part or this sequence of parts. Software agent would take a part name and using the ontology definitions would query other registries via their semantic web interfaces (no need to know about schema: e.g., just say "need all <#part>s that match a <#component> of the given <#part>"). Software agent can search anyone's registry if they use a common ontology: simply follow URLs (or use query language) and add triples to the local RDF store. *Superpart Search: search for parts that contain the given parts *What about sub- and superpart searches in distributed registries? *Search for function (case insensitive): repressor, reporter, inverter, etc. *What are the available (instances of) parts? Are they used in any devices already? (saves time for constructing expression device). Problem: different names for exactly same DNA sequence *What kinds of devices/systems have been built? *Search for "similar" parts *Find all parts of the form promoter.Q04400 where promoter can be any promoter. It would also be nice to specify whether you want any parts, just available parts or available and working parts. ==Miscellaneous use cases== *Find a low-copy vector (1 to 2 per cell) which carries two antibiotic marker genes.
